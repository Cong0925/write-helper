# 写作助手 — 产品需求文档

## 1. 项目概述

一款面向小说创作者的本地化写作管理工具，帮助作者管理小说项目的文件结构、撰写内容、组织素材。

- **目标平台**：Windows 桌面端
- **技术栈**：Tauri 2.0 + Rust 后端 + 前端（待定）
- **核心原则**：离线运行，无需登录，无需联网，本地文件即数据
- **项目代号**：writeHelper

---

## 2. 技术选型

| 层 | 技术 | 原因 |
|---|---|---|
| 桌面框架 | Tauri 2.0 | 体积小（<10MB）、性能好、 Rust 安全 |
| 后端语言 | Rust | Tauri 原生 |
| 前端框架 | Vue 3 | 已确定 |
| 数据存储 | 文件系统（Markdown + JSON） | 用户直接可读，无数据库依赖 |
| 项目标识 | `.writinghelper/project.json` | 识别由本软件创建的项目 |

---

## 3. 核心功能

### 3.1 项目管理

- **新建项目**：用户选择一个空文件夹，软件自动创建标准化写作目录结构
- **打开项目**：只能打开由本软件创建的项目（通过 `.writinghelper/project.json` 识别）
- **项目元数据**：在 `.writinghelper/project.json` 中存储项目名称、创建时间、最近打开时间

### 3.2 目录结构

新建项目时自动生成以下结构：

```
项目根目录/
├── .writinghelper/
│   └── project.json              # 项目元数据（标识是本软件创建）
├── 大纲/
│   ├── 主线大纲.md
│   └── 分卷大纲.md
├── 分卷/
│   ├── 第一卷/
│   │   ├── 第一章.md
│   │   └── 第二章.md
│   └── 第二卷/
├── 人设/
│   ├── 主角.md
│   ├── 配角/
│   └── 反派.md
├── 设定/                         # 世界观/设定
│   ├── 地理.md
│   ├── 历史.md
│   ├── 势力.md
│   └── 特殊规则.md
├── 素材/                         # 素材收集
└── 输出/                         # 导出目录
```

- 用户可以手动在此结构下创建新的文件或文件夹
- 左侧导航树与文件系统保持同步

### 3.3 编辑器

- Markdown 编辑器，支持基本格式（标题、粗体、斜体、列表）
- 分卷章节支持前后章节快速切换
- 自动保存（防丢稿）

### 3.4 导航树

- 左侧树状视图展示完整项目结构
- 左键点击节点 → 在右侧打开文件
- 右键节点 → 上下文菜单（新建文件、新建文件夹、重命名、删除）
- 目录结构与文件系统实时同步（用户在文件系统中的操作反映到树上）

### 3.5 大纲视图

- 解析 Markdown 标题结构，生成可折叠的大纲面板
- 点击标题跳转到对应位置

### 3.6 导出功能

- 支持将分卷/全书导出为单一 Markdown 文件
- 导出到 `输出/` 目录

## 5. 用户流程

### 5.1 首次使用 / 新建项目

1. 启动应用 → 显示欢迎界面（新建项目 / 打开项目）
2. 点击「新建项目」→ 选择文件夹 → 输入项目名称
3. 应用创建目录结构 → 进入主界面
4. 自动打开 `大纲/主线大纲.md` 供编辑

### 5.2 打开已有项目

1. 启动应用 → 欢迎界面点击「打开项目」
2. 选择包含 `.writinghelper/` 的项目根目录
3. 应用验证项目合法性 → 进入主界面
4. 恢复上次关闭时的状态（如上次编辑位置）

### 5.3 日常写作

1. 打开项目 → 左侧导航树找到章节
2. 点击章节 → 右侧打开编辑器
3. 写作 → 自动保存
4. 可通过大纲视图跳转章节内位置

### 5.4 新建章节 / 文件

1. 在导航树中右键目标目录（如「第一卷」）
2. 选择「新建文件」→ 输入文件名
3. 空白文件创建完成 → 在编辑器中打开

## 6. 非功能性需求

- **启动速度**：冷启动 < 2 秒
- **包体积**：安装包 < 20MB
- **内存占用**：空闲时 < 100MB
- **文件编码**：统一 UTF-8
- **安全性**：不收集任何用户数据，不上传任何文件

## 7. 开发路线

### Phase 1 — 项目骨架
- Tauri 项目初始化
- 前端框架选型与搭建
- 目录结构自动生成逻辑
- 项目新建 / 打开基础流程

### Phase 2 — 导航与编辑
- 左侧导航树实现（文件系统遍历）
- Markdown 编辑器集成
- 右键上下文菜单
- 自动保存

### Phase 3 — 进阶功能
- 大纲视图（Markdown 标题解析）
- 字数统计
- 导出功能
- 暗色/亮色主题切换

### Phase 4 — 打磨
- 启动画面 / 欢迎页
- 会话恢复（记住上次打开的项目和位置）
- 性能优化
- 安装包打包

### Phase 5 — AI 助手（可选扩展）
- 用户填写自有 API Key（如 DeepSeek）存储在本地配置中
- 选中文本后通过右键或侧边栏唤起 AI 功能
- Rust 后端通过 `reqwest` 调用第三方 API（兼容 OpenAI 格式）
- 对话历史存储在本地项目目录中
- 全部请求直连第三方接口，不经过任何中间服务器
