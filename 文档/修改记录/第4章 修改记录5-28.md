一、底层架构：两个 “编辑区” 叠在一起
页面上其实放了两个东西，但同一时间只显示一个：
可视化层（WYSIWYG）
一个带 contenteditable="true" 的 <div>（或 iframe）
你看到的 “所见即所得” 排版
内部是真实 DOM 节点（<div> <p> <strong> <style>…）
对应：平时你编辑文章的样子
代码层（HTML 源码）
一个 <textarea> 或代码编辑器（带高亮）
显示一长串文本：就是 HTML 源代码字符串
没有 DOM，只有纯文本字符串
对应：点 HTML 按钮后看到的代码
布局上：两个容器完全重叠，CSS 控制谁 display: block、谁 display: none。
二、点击「HTML」按钮时发生了什么（核心流程）
1）可视化 → 代码（第一次点）
JS 执行：html = 可视化层.innerHTML
→ 把当前所有 DOM 转成HTML 字符串
