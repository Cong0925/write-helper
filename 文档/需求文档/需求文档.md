# 写作助手 — 产品需求文档

## 1. 项目概述

一款面向小说创作者的本地化写作管理工具，帮助作者管理小说项目的文件结构、撰写内容、组织素材、校对文本、导出作品。

- **目标平台**：Windows 桌面端（跨平台支持 macOS / Linux）
- **技术栈**：Tauri 2.0 + Rust 后端 + Vue 3 前端
- **编辑器**：CodeMirror 6（Markdown）+ 所见即所得编辑器（人设/设定等面板）
- **核心原则**：离线运行，无需登录，无需联网，本地文件即数据
- **项目代号**：writeHelper
- **应用名称**：写作助手
- **当前版本**：v0.1.0

---

## 2. 技术选型

| 层 | 技术 | 原因 |
|---|---|---|
| 桌面框架 | Tauri 2.0 | 体积小（<10MB）、性能好、Rust 安全 |
| 后端语言 | Rust | Tauri 原生 |
| 前端框架 | Vue 3 + TypeScript | 已确定 |
| 编辑器 | CodeMirror 6 | 轻量、稳定、原生 Markdown 体验 |
| Markdown 渲染 | marked + TurndownService | 所见即所得编辑器使用 |
| 数据存储 | 文件系统（Markdown + JSON） | 用户直接可读，无数据库依赖 |
| 项目标识 | `.writinghelper/project.json` | 识别由本软件创建的项目 |
| 打包 | Tauri NSIS（Windows 安装包）| |

## 3. 核心功能

### 3.1 项目管理

- **新建项目**：用户选择保存位置，输入项目名称和简介，自动创建标准化写作目录结构
- **打开项目**：只能打开由本软件创建的项目（通过 `.writinghelper/project.json` 识别）
- **导入项目**：支持导入 .txt、.md、.docx 格式的小说文件，自动识别章节和分卷结构并拆分导入
- **最近项目列表**：显示最近打开的项目（最多 10 个），支持重命名、删除（含项目文件夹）、在文件管理器中定位
- **项目元数据**：在 `.writinghelper/project.json` 中存储项目名称、描述、封面路径、创建时间、最近打开时间

### 3.2 目录结构

新建项目时自动生成以下结构：

```
项目根目录/
├── .writinghelper/
│   └── project.json              # 项目元数据（标识是本软件创建）
├── 大纲/
│   └── 主线大纲.md               # 初始大纲文件
├── 分卷/
│   └── 第1卷/
│       └── 第1章.md              # 初始章节
├── 人设/                         # 角色管理
├── 设定/                         # 世界观/设定
├── 素材/                         # 素材收集
└── 输出/                         # 导出目录
```

- 用户可以手动在此结构下创建新的文件或文件夹
- 左侧导航树与文件系统保持同步
- 支持无限层级的子目录，但默认深度限制为 5 层

### 3.3 编辑器

**主编辑器（CodeMirror 6）:**

- Markdown 语法高亮
- 行号显示
- 自动换行
- Tab 缩进
- 分卷章节标题显示（如"第1卷 / 第1章"）
- 点击标题可编辑（自动同步文件名）
- 自动保存（默认开启，可关闭，防丢稿）
- 手动保存快捷键 Ctrl+S
- 字数统计（总字符、中文字数、行数）
- 编辑器右键菜单（复制、剪切、粘贴、全选）
- 一键排版（Ctrl+K）：修剪行首尾空格、合并空行、重排列表编号、统一半角标点为全角、中英文间加空格、首行缩进

**所见即所得编辑器（WysiwygEditor）:**

- 用于侧边面板（人设/设定/素材/大纲）的内容编辑
- 支持标题 H1-H3、粗体、斜体、无序/有序列表、引用、行内代码
- 粘贴纯文本（自动去除格式）
- 底层使用 Markdown 格式存储（通过 TurndownService 转换）

### 3.4 左侧面板

**搜索栏：**
- 全书搜索，输入后 2 秒防抖自动搜索
- 高级选项：区分大小写、全词匹配、正则表达式
- 搜索结果按文件分组，显示匹配行预览，高亮关键字
- 点击匹配行跳转到对应文件和行号

**分卷/章节树：**
- 树状视图展示分卷和章节
- 分卷按自然数排序（第2卷 < 第10卷）
- 自动展开所有节点
- 卷操作：新建章节、重命名、合并导出、删除整卷
- 章节操作：重命名、删除
- 章节间拖拽移动（跨卷拖动）
- 底部的"新建章节"按钮
- 新建章/卷时自动计算编号

**动作按钮：**
- ＋新建章
- ＋新建卷
- 刷新目录

### 3.5 右侧面板

**侧边栏按钮：**
- 校对（✓）
- 大纲（☰）
- 人设（👤）
- 设定（🌍）
- 素材（📎）

**校对面板：**
- 错别字检测：内置 ~550 条常见错别字词典（"的/地/得"、"在/再"、"已/以/己"等分类）
- 风险检测：用户自定义敏感词检测
- 自定义纠错：用户可配置错别字典
- 病句检测（占位，需接入 AI 模型后可用）
- 每个检测项可：跳转到位置、应用修正、忽略（单次/永久）
- 错别字支持"全部修改"
- 可折叠的模块化设计

**大纲面板（FolderPanel）：**
- 展示 `大纲/` 目录下的文件
- 支持文件搜索、新建、重命名、删除、上移/下移排序
- 右侧分栏：文件列表 + 内容预览（WysiwygEditor）

**人设面板（CharacterPanel）：**
- 展示 `人设/` 目录下的文件
- 支持分类夹 + 角色文件
- 创建新角色时自动生成模板（角色名、背景、性格、外貌等）
- 文件名与角色名自动同步（编辑第一行后 1.5 秒自动重命名文件）
- 支持切换分类、跨分类移动角色
- 搜索、重命名、删除、上移/下移

**设定面板、素材面板：** 同 FolderPanel，分别对应 `设定/` 和 `素材/` 目录

### 3.6 查找替换（浮动面板）

- 可拖动的浮动窗口
- 查找/替换两个标签页
- 查找输入、替换输入、匹配计数
- 范围：当前章节 / 全书
- 高级选项：区分大小写、全词匹配、正则表达式
- 上一个/下一个导航
- 替换当前 / 全部替换
- 匹配结果列表，点击跳转

### 3.7 导出功能

- **合并导出**：选择章节（支持全选、按卷选择），合并为一个文件
- **导出格式**：Markdown (.md)、纯文本 (.txt)、HTML (.html)
- **合并预览**：实时预览合并后的内容
- **导出方式**：通过系统保存对话框选择输出位置

**快捷键一览：**

| 全屏 | Ctrl+. |

**文件位置：**
- 默认项目目录（新建项目时自动填充）

**其他设置：**
- 自动保存开关
- 开机自启动（Windows 注册表）

### 3.9 字体设置（浮动面板）

- 字体选择（微软雅黑、宋体、黑体、楷体、仿宋、JetBrains Mono、Consolas 等）
- 字号滑块（10-24）

## 5. 用户流程

### 5.1 首次使用 / 新建项目

1. 启动应用 → 显示欢迎界面（左侧：新建项目 / 打开项目 / 导入项目；右侧：最近项目列表）
2. 点击「新建项目」→ 输入作品名称、简介、选择保存路径
3. 应用创建标准目录结构 → 进入主界面
4. 自动打开 `大纲/主线大纲.md` 供编辑（实际当前行为是进入主界面等待用户操作）

### 5.2 打开已有项目

1. 启动应用 → 欢迎界面点击「打开项目」
2. 选择包含 `.writinghelper/` 的项目根目录
3. 应用验证项目合法性 → 进入主界面

### 5.3 导入项目

1. 欢迎界面点击「导入项目」
2. 选择 .txt / .md / .docx 文件
3. 自动识别分卷（第X卷）和章节（第X章）结构
4. 预览识别结果（分卷列表、章节预览）
5. 输入项目名称 → 点击导入
6. 自动创建标准项目结构并拆分章节文件
7. 进入主界面

### 5.4 日常写作

1. 打开项目 → 左侧导航树找到章节
2. 点击章节 → 右侧打开 CodeMirror 编辑器
3. 写作 → 自动保存（1.5 秒防抖）
4. 可通过右侧边栏的校对功能检查错别字
5. 可通过查找替换功能进行全书搜索和替换

### 5.5 新建章节 / 分卷

1. 在左侧面板点击「＋新建章」或「＋新建卷」
2. 自动按顺序编号（第N章 / 第N卷）
3. 新文件创建完成 → 在编辑器中打开
4. 可点击标题区域重命名

### 5.6 管理角色

1. 点击右侧边栏「人设」按钮
2. 在右侧面板中新建分类（如"主角"、"配角"）
3. 在每个分类下创建角色文件
4. 使用 WysiwygEditor 编辑角色信息
5. 修改第一行角色名，文件名自动同步

### 5.7 导出

1. 在左侧面板分卷的右键菜单选择「合并导出章节」
2. 勾选要导出的章节（支持按卷全选）
3. 选择导出格式（Markdown / 纯文本 / HTML）
4. 预览合并效果
5. 选择保存位置 → 导出完成

## 8. 校对准系统

### 8.1 错别字检测
- 内置词典，按类别组织：的/地/得（57条）、在/再（13条）、已/以/己（18条）、做/作（22条）、即/既（9条）、成语/常见错别字（约430条）
- 每一条目为 wrong → correct 的映射
- 检测策略：最长匹配优先，按行扫描
- 同一位置去重

### 8.2 敏感词检测
- 用户自定义敏感词列表
- 支持模板导入（批量添加）
- 持久化存储

### 8.3 自定义纠错
- 用户可创建多个纠错规则组
- 每组可独立启用/禁用
- 支持替换或删除操作

### 8.4 格式检查
- 检查中文标点使用规范
- 其他格式问题检测

### 8.5 忽略机制
- 单次忽略（会话级别）
- 永久忽略（持久化到文件，基于文件路径+文本内容+行内容三重标识）

## 9. 非功能性需求

- **启动速度**：冷启动 < 2 秒
- **包体积**：安装包 < 20MB
- **内存占用**：空闲时 < 100MB
- **文件编码**：统一 UTF-8
- **安全性**：不收集任何用户数据，不上传任何文件
- **编辑器性能**：支持万行以上文档流畅编辑（CodeMirror 6）
- **搜索性能**：全书搜索 2 秒防抖，避免频繁磁盘 IO
- **皮肤系统**：纯 CSS 变量驱动，无重绘闪烁
